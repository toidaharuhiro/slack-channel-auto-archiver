# コンテキストウィンドウ管理

Claude Code CLI上でトークン消費を抑えるためのルール。

## ファイル読み込みの原則
- CLAUDE.md は常にコンテキストに入る → 200行以内を厳守
- `docs/ARCHITECTURE.md`, `skills/*.md` は必要時のみ読む。不要なら読まない
- ソースファイルは変更対象のみ読む。全ファイル一括読み込みしない
- テストファイルは対応するソースと一緒にのみ読む

## 応答の原則
- 不要な説明・前置きを省き、コード変更と結果に集中する
- 変更箇所のdiffのみ示す。変更しないファイルの内容を繰り返さない
- ハーネス実行結果は成否サマリのみ報告。全出力を貼らない

## ファイル作成・編集の原則
- 新規ファイルは最小限の内容で作成し、必要に応じて拡張
- 大きな変更は複数の小さなコミットに分割
- リファクタリングと機能追加は別の作業として実施
